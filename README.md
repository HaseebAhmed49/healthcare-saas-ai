# Healthcare SaaS AI

A Next.js app for licensed healthcare providers to generate AI-powered consultation summaries, next steps, and patient-friendly email drafts from doctor notes.

## Description

This app combines a React/Next.js frontend with a Python-based FastAPI backend. Authenticated users submit patient visit notes, and the backend streams a structured AI summary generated by OpenAI.

Key features:
- Secure Clerk authentication for provider access
- Premium plan gating via Clerk subscriptions
- Real-time SSE streaming of AI-generated consultation summaries
- FastAPI Python backend deployed under `/api`
- Markdown-friendly output rendering on the frontend

## App Workflow

1. Provider signs in using Clerk and accesses the consultation form on `pages/product.tsx`.
2. Provider enters:
   - patient name
   - visit date
   - consultation notes
3. The frontend sends a POST request to `/api` with the provider's Clerk JWT token.
4. The Python backend in `api/index.py` verifies the bearer token using Clerk and then calls OpenAI with a structured prompt.
5. The AI response is streamed back as server-sent events (SSE).
6. The React client gradually appends and renders the summary in Markdown.

## Project Structure

- `pages/product.tsx` — main consultation workflow with Clerk auth and SSE streaming
- `api/index.py` — FastAPI endpoint for AI summarization and Clerk validation
- `requirements.txt` — Python dependencies for the API runtime
- `package.json` — Next.js app dependencies
- `next.config.ts` — Next.js configuration
- `pages/index.tsx` — homepage for the app

## Local Development

### Prerequisites

- Node.js 20+ / npm
- Python 3.11+ (for the Python API)
- Vercel CLI if you want to deploy from the command line

### Install dependencies

```bash
cd healthcare-saas-ai
npm install
pip install -r requirements.txt
```

### Environment variables

Create a `.env.local` file at the project root with:

```bash
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=<your-clerk-publishable-key>
CLERK_SECRET_KEY=<your-clerk-secret-key>
CLERK_JWKS_URL=<your-clerk-jwks-url>
OPENAI_API_KEY=<your-openai-api-key>
```

If you also use Clerk redirects or other Clerk settings, set those values in `.env.local`.

### Configure Clerk

1. Sign in to the Clerk dashboard at https://clerk.com and create a new application.
2. In your Clerk application settings, copy:
   - `Publishable Key` → `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`
   - `Secret Key` → `CLERK_SECRET_KEY`
3. Under `API Keys` or `JWKS`, copy the application JWKS URL to `CLERK_JWKS_URL`.
4. Configure Clerk redirect URLs to include your local development URL, for example:
   - `http://localhost:3000/*`
   - `https://<your-vercel-app>.vercel.app/*`
5. Ensure Clerk session and authentication settings are enabled for your Next.js app.

### Run locally

```bash
npm run dev
```

Open `http://localhost:3000` in your browser.

## Vercel Deployment

### 1. Verify the Python API configuration

Ensure your Python backend dependencies are listed in `requirements.txt` and not misspelled.

### 2. Add required environment variables in Vercel

Set these in your Vercel project settings:
- `CLERK_JWKS_URL`
- `OPENAI_API_KEY`

### 3. Connect the repository to Vercel

1. Go to [vercel.com](https://vercel.com).
2. Import your GitHub repository `HaseebAhmed49/healthcare-saas-ai`.
3. Choose the `main` branch.
4. Confirm settings.

### 4. Deploy

Click `Deploy` in the Vercel dashboard, or use the CLI:

```bash
vercel --prod
```

### 5. Confirm the deployment

- Application root should load successfully.
- The `/api` endpoint should respond correctly for authenticated requests.

## Troubleshooting

- If the API returns `500`, confirm `requirements.txt` exists and can be installed.
- If authentication fails, verify `CLERK_JWKS_URL` is correct and Clerk is configured correctly.
- If OpenAI calls fail, confirm `OPENAI_API_KEY` is valid and available to the deployment.

## Notes

- This app is designed for a secure provider experience: only authenticated users with the premium plan view the consultation form.
- Changes to the form or prompt should be made in `pages/product.tsx` and `api/index.py` respectively.
